BryTorch tests

BryTorch runs the real PyTorch in the browser — the genuine torch._C (whole-archive ATen compiled to WebAssembly) plus torch's own pure-Python layer on top of Brython, through the Wasthon C-API bridge. Not a reimplementation: upstream PyTorch itself.

Test-suite dashboards

  • ▶ PyTorch suite dashboard 1/2 — PyTorch's own test/test_*.py (with the real torch.testing._internal harness) against the wasm torch._C: test_torch then the quick suites, two iframes so each stage runs on a fresh heap.
  • ▶ PyTorch suite dashboard 2/2 — the long suites, same two-iframe scheme: test_autograd + half of test_reductions, then the other half.
  • ▶ CPython suite dashboard — CPython's own Lib/test/test_*.py against the Wasthon bridge stack BryTorch runs on (the authoritative behavioural reference).

Smoke page

  • ▶ BryTorch — boots torch._C, imports torch end to end, then tensor create / + / * / @ / sum() with the real ATen kernels.
